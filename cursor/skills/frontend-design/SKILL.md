# Frontend Design Skill
Skill for designing and implementing user interfaces and frontend components.

## When to Use
- When implementing UI components from designs/mockups.
- When creating new pages or views.
- When improving user experience.
- When implementing responsive layouts.

## Instructions
- **Design Principles:**
    1. Follow the existing design system/style guide.
    2. Prioritize accessibility (a11y).
    3. Ensure responsive design for different screen sizes.
    4. Maintain consistent UX patterns.
- **Implementation:**
    1. Use semantic HTML.
    2. Follow component-based architecture.
    3. Separate concerns (structure, style, behavior).
    4. Optimize for performance (lazy loading, code splitting).
- **Styling:**
    1. Use project's CSS methodology (BEM, CSS Modules, Tailwind, etc.).
    2. Maintain consistent spacing, colors, and typography.
    3. Test across browsers if required.
- **Output:**
    - Component files (JSX/TSX, Vue, etc.).
    - Style files (CSS/SCSS/styled-components).
    - Documentation for reusable components.
- **Constraints:**
    - Match designs/mockups accurately.
    - Ensure cross-browser compatibility.
    - Follow accessibility guidelines (WCAG).
