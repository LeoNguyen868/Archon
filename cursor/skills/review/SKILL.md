# Review Skill
Skill for reviewing code and documents to ensure quality, security, and performance.

## When to Use
- When Dev completes implementation (Code Review).
- When Tech Consultant completes Tech Spec (Design Review).
- When PM completes Plan (Plan Review).
- Before merging or closing a task.

## Instructions
- **Code Review:**
    - Clean Code: Naming, formatting, complexity (KISS).
    - Logic: Correctness, bug detection.
    - Security: Input validation, data protection, no secrets exposed.
    - Performance: No obvious bottlenecks.
    - Tests: Unit tests adequate and passing.
- **Document Review:**
    - Clarity: Clear and easy to understand.
    - Completeness: Covers all required sections.
    - Consistency: Consistent with other documents.
- **Output:** Structured feedback with severity levels (Critical, Major, Minor, Nit).
